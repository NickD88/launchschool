Block Elements

Block level elements take up the entire width of the page and always start a new line. It can have margin/s and padding/s. If no width is set it will expand naturally to fill its parent container. If no height is set, it will expand to fit its content.

Example: (Div elements are naturally block level elements)

This is a div with no width and height set.
This is a div with no width but with a height of 60px, with a padding and margin of 20px around it.
This is a div with a width of 90px and no height set

In the first example, the div fills the entire width, and uses just enough height to fill its content. In the second box, we didn't set a width, but it still filled the entire width (counting the pading and margin); it also uses 60px of height plus the margin and padding. For the third box, we set the width to 90px, as expected, it took just enough height to fill its content. The text was squished because of the reduced width, so the browser expanded the height to accommodate the content.

Positioning

Every now and then we want to position things the way we want it to be and move it around the page, the floats simply cannot do this as it can only take an element to position left or right of the page. This is where positioning comes in. There are four tpes of positioning:

  • Static
  • Fixed
  • Relative
  • Absolute

Static Position

The element renders normally on the page - this is the default position of elements on the page.

Fixed Position

This will allow you to put an element in the same place even when the page is scrolled. Top, Right, Bottom and Left properties are used to position this property

Fixed Position Elements are taken out of the flow and does not leave a gap in the location of the page where you originally put it.

notice the blue box on the bottom left side of the page. It is a fixed position element.

I'm positioned fixed

Position Relative

Positioned relative to its normal position. Setting the offset values will cause the elements to move away from its normal position acording to the values you've set. However, its position in the normal flow of the document will be preserved and folling texts will not wrap around it.

When we position an element with offset values it moves ton top of the next text/elements rather then pushing it down as to what padding and margin element would do.

When the element is positioned absolut, it is taken out of the normal flow and the original space of the element in the document will not be preserved.

Absolutely positioned elements are moved in relation to their closest relitavely positioned prent element. Should a relatively positioned parent element not exist, the absolutely positioned element wil be positioned in relation to the <body> element.
